# Outreach Packets v2

## Scope

Packet set aligned to Exposure Plan v2:
- Tech partner packet
- Cultural partner packet
- City/Institution packet

Use project-specific asks and avoid generic umbrella messaging.

## Shared Intro Block (Use in all packets)

We operate project-first.  
Public tracks: Studio Pass, Dimensional Knob, Fusion Factory.  
Scouting Label is currently private-deferred pending framing/legal hardening.  
Cross-project references are included only where they directly improve ROI/risk clarity.

## Packet A: Tech Partner

Target examples:
- Ableton
- Native Instruments
- Serato
- Pioneer
- Steinberg
- FL Studio ecosystem-relevant channels

Purpose:
- integration credibility,
- compatibility narratives,
- technical sponsorship or pilot stack support.

Required sections:
1. Project context
2. Why this partner specifically
3. Concrete integration/sponsorship ask
4. Pilot proof target and timeline
5. Mutual upside and next-step decision

## Packet B: Cultural Partner

Target examples:
- Boiler Room
- EXIT Festival
- comparable festival/media channels

Purpose:
- amplification,
- program co-production,
- audience and artist-channel expansion.

Required sections:
1. Program concept fit
2. Cultural relevance and audience value
3. Co-production ask
4. Documentation/capture plan
5. Pilot event output definition

## Packet C: City/Institution

Target examples:
- city cultural departments,
- venue controllers,
- municipal innovation/culture offices,
- public infrastructure stakeholders.

Purpose:
- venue access pathway,
- compliance and program legitimacy,
- pilot MoU pathway.

Required sections:
1. Civic value proposition
2. Safety/compliance readiness baseline
3. Residential impact mitigation outline
4. Pilot timeline and operating model
5. Concrete ask: hall access + pilot window + operating coordination

## Quality Rules

1. No generic mass email tone.
2. Ask must be concrete and measurable.
3. Claims must be evidence-backed or explicitly marked as pending pilot proof.
4. Do not expose Scouting Label broadly in this cycle.
5. Include provenance/IP block when sending project-facing materials.
